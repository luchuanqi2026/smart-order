# AI 考试题目: 万能导入 V2

## 智能多格式批量下单系统 —— 大模型 + 钉钉预警 + 高性能

---

## 一、项目背景

物流/快递行业需要频繁批量下单，客户提供的文件格式各异（Excel / Word / PDF），且文档结构复杂（有干扰性头部信息、横向排列字段、合并单元格、非标准表格等）。现需开发一个 Web 应用，通过接入大模型（LLM）实现任意格式文件的智能解析与导入，完成批量下单流程，并部署到 Vercel 平台在线访问。

**核心变化（相较 V1）：**
1. UI 风格须与鲸天系统保持一致
2. 通过大模型实现任意格式解析
3. 严肃性能要求

## 三、下单字段定义

> 说明：每条出库单根据外部编码聚合，同一个外部编码下的多个 SKU 行共享一组收货信息，展示为一个出库单。

| 字段名 | 说明 | 必填 |
|--------|------|------|
| 外部编码 | 外部系统订单唯一编号，用于去重和聚合（如配送单号） | 是 |
| 收货门店 | 收货门店/机构名称，如"尹三顺自助烤肉(银泰店)" | A组 |
| 收件人姓名 | 收货人姓名 | B组 |
| 收件人电话 | 收货人联系方式 | B组 |
| 收件人地址 | 收货人完整地址 | B组 |
| SKU物品编码 | SKU 唯一编码 | 是 |
| SKU物品名称 | SKU 名称 | 是 |
| SKU发货数量 | 发货数量，必须为正数 | 是 |
| SKU规格型号 | 物品规格描述 | 否 |
| 备注 | 附加说明 | 否 |

**A组 vs B组（二选一）：**
- **A组（门店模式）**：只需填写"收货门店"，不要求收件人姓名/电话/地址
- **B组（收件人模式）**：需填写"收件人姓名 + 收件人电话 + 收件人地址"三个字段，不要求收货门店

两组都填也可以，但至少填一组。两组都没填则校验不通过。

## 四、功能需求

### 模块一：解析规则配置 + AI 辅助生成（核心考点）

随题附 4 份真实出库单文件，格式和业务结构完全不同。**严禁为每种文件硬编码解析逻辑**，系统必须提供可配置的解析规则体系，并且支持通过 AI 大模型自动分析文件并生成解析规则。

**核心设计理念：** 不是写 4 个 if/else 去适配 4 种文件，而是设计一套通用的规则描述语言/规则引擎，每种新格式只需要"配置一条规则"即可适配。规则可以由用户手动配置，也可以由 AI 分析文件后自动生成。理想状态下，新增第 5 种、第 10 种格式的出库单时，系统代码零改动。

- **解析规则管理：** 规则保存到服务器端持久化存储，支持创建、编辑、删除、复制。上传文件时由用户手动选择使用哪条规则，不做自动匹配。提供"新建规则"按钮，用户可随时创建新规则

- **AI 辅助生成规则（关键能力）：** 导入文件后，先由大模型预处理——AI 分析文件结构并生成一套推荐规则，然后用户在此基础上手动微调确认后保存。AI 需明确标注哪些映射是推测的，让用户做最终确认

- **规则预览测试：** 保存规则前，可用当前文件作为样例进行试解析，实时预览解析结果，确认正确后再保存

规则的具体结构如何描述、字段如何映射、复杂格式如何处理，由考生根据 demo 文件的实际差异自行设计方案。

#### 出库单差异说明

> 注意：以下文件为考核用例，考生不得为任何一份文件编写专门的硬编码解析逻辑。所有兼容能力必须通过配置不同的解析规则来实现。代码中不应出现任何文件名判断或特定列名硬编码。

| 文件 | 格式 | 结构特征 | 核心难点 |
|------|------|----------|----------|
| 配送单1 | Excel | 干扰头部（前6行非数据），第7行合计，第8行底部元信息，第9-10行无用信息 | 头部跳过、尾部提取 |
| 配送单2 | Excel | 32列，第1行说明文字，2行表头，第3行起数据 | 复杂表头、宽表格 |
| 湖南客户发货明细 | Excel | 167行，每行包含收货人/电话/地址 + SKU 明细 | 按配送单分组，同一单内的多个物品行共享同一收货信息 |
| 生鲜配送单 | Excel | 完全不同的业务模型：SKU 横向展开，门店名作为列头横向排列 | 矩阵转置 |
| 点菜单 | PDF | 头部元信息，中部标准表格，底部签字区，收货人/电话/地址在最后一页底部 | 跨页提取底部信息、合计行过滤 |
| 多门店分单 | Excel | 多个 Sheet，每个 Sheet 是一个独立门店的出库单，结构相同但数据不同 | 多 Sheet 合并，每个 Sheet 底部有横向排列的收货人信息 |
| 卡片式单据 | Excel | 非标准表格，"卡片"区域，每个卡片内部有独立表头和收货信息 | 卡片式拆分 |
| 纯文本单据 | Word | 纯文本段落格式，无任何表格。每条记录用"编号:值"格式嵌在段落中 | 文本段落解析、非结构化 |
| 复合单元格单据 | Excel | 日期作为列头横向展开，单个单元格内多行文本需拆为多行 | 复合单元格拆分 |
| PDF多订单 | Excel | 一个 PDF 内含 3 个独立的配送签收单，每个有独立收货人信息和物品明细表格 | PDF 多订单拆分 |

### 模块二：文件导入与解析执行

- 支持上传 Excel（.xlsx/.xls）、Word（.docx）、PDF 文件，支持拖拽上传和点击上传

- **解析流程：** 上传文件 → 用户手动选择已有规则或点击"新建规则" → 若新建则大模型先预处理分析文件生成推荐规则 → 用户手动微调确认 → 执行解析 → 展示结果

- 所有规则匹配均为手动操作，不做自动匹配。用户从规则列表中选择或新建

- 导入时显示实时进度条，显示百分比和当前处理条数/总条数

- 完善的错误处理：文件格式错误、文件为空、编码异常、无法解析等情况需给出明确提示

- 解析失败时，需给出明确提示并将原始文件信息展示给用户，并提供手动配置规则的入口

---

### 模块三：数据预览与编辑

- 导入后进入预览列表页面，以类 Excel 表格形式展示所有数据
- 表头固定、支持横向滚动、单元格点击可直接编辑（类似在线 Excel）

- **行内错误实时校验：** 必填字段缺失标红、电话格式错误标红、数量非正数标红
- **全部错误一次性展示：** 校验时必须同时列出所有错误，明确标注具体行数、字段名称和错误原因

- **外部编码重复检测：** 同批次内重复 + 与已存在数据重复，均在对应行高亮提示
- 支持删除行、新增空行操作
- 支持将预览数据（含编辑）导出为 Excel 文件

---

### 模块四：提交下单

- 有错误的行不允许提交，需提示用户先修正
- 点击"提交下单"后显示上传进度条
- 提交成功后数据持久化到数据库
- 返回提交结果汇总：成功 N 条、失败 N 条

---

### 模块五：已导入运单列表

- 可查看所有历史已导入的运单记录（从数据库读取）
- 支持按外部编码、收件人姓名、提交时间进行筛选/搜索
- 支持分页展示

## 七、附：测试文件说明

随题附多种格式的复杂测试文件，用于验证大模型智能解析能力。

### 格式覆盖
- **Excel：** 干扰头部 + 散落尾部 + 跨行聚合 + 矩阵转置 + 多Sheet + 卡片式 + 复合单元格
- **Word：** 纯文本段落式（无表格），物品信息嵌在文本行中
- **PDF：** 干扰头部 + 标准表格 + 底部签字区 + 独立签收页

### 难度特征
- **干扰头部：** 文件开头包含公司名称、日期、业务状态、配送信息等大量非数据字段
- **尾部散落信息：** 收货人/电话/地址在数据区之外的独立行或文件末尾
- **跨行聚合：** 同一配送单号下多行物品共享一组收货人信息，需按单号聚合
- **矩阵转置：** 门店名/日期作为列头横向排列，需转置为独立运单记录
- **多Sheet合并：** 每个Sheet是一个独立门店的出库单，需遍历所有Sheet合并解析
- **卡片式堆砌：** 每条记录是一个独立的"卡片"区域，非标准表格行，需按卡片边界拆分
- **纯文本解析：** Word文档无任何表格，物品信息用"编号:值"格式嵌在段落中
- **复合单元格：** 一个单元格内含"物品名x数量\n物品名x数量"的复合值，需拆分为多行
- **PDF多独立订单：** 同一PDF内含多个独立签收单，每单有自己的收货人和物品明细
